Frequently Asked Questions

How do I know if something is a bug?

A bug is when the software doesn't work as expected or described in the documentation. This includes crashes, errors, incorrect calculations, or any unexpected behavior that prevents you from using a feature properly.

What information should I include in a bug report?

The most helpful bug reports include: clear steps to reproduce the issue, what you expected to happen, what actually happened, your software version, operating system, and screenshots if applicable. The more details you provide, the faster we can fix the issue.

How are feature requests prioritized?

We evaluate feature requests based on several factors: how many users would benefit, technical complexity, alignment with our product roadmap, and potential business value. While we can't implement every suggestion, we read and consider all feedback.

Will I receive updates about my bug report or feature request?

If you provide your email address, we'll notify you of significant updates related to your submission. For bugs, we'll let you know when they're fixed. For feature requests, we'll inform you if we decide to implement them in a future release.

I found a security vulnerability. Should I report it here?

No. For security vulnerabilities, please email us directly at security@argorobots.com instead of using this form. This helps us address sensitive security issues appropriately.

How can I check if my bug or feature request has already been reported?

We're working on a public issue tracker that will allow users to see existing reports. In the meantime, you can contact our support team at support@argorobots.com to check if your issue has already been reported.
